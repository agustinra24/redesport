Práctica 5: Modelo OSI y TCP_IP

Cristobal Lopez Apango

Ing. Mecatrónica

Resumen

Las redes de computadoras son fundamentales en todos los ámbitos, desde las más simples hasta las más avanzadas, permitiendo la comunicación global en diversas formas. En esta práctica se abordaron las arquitecturas de red más utilizadas, explorando su funcionamiento y configuración. Además, se estudiaron los componentes esenciales de una red local (LAN), proporcionando una comprensión integral de cómo estas redes se establecen y operan.

Introduccion

Las redes de computadoras son fundamentales en la infraestructura tecnológica de la era digital, proporcionando la base para la conectividad global. A través de estas redes, se posibilita la comunicación y el intercambio de datos entre diversos dispositivos, habilitando una multitud de aplicaciones y servicios que son esenciales en nuestra vida diaria y profesional. Es crucial entender la relevancia de las redes y sus componentes físicos para poder diseñar, implementar y mantener sistemas de comunicación robustos y seguros en distintos entornos. Entre los componentes físicos de una red se encuentran dispositivos como routers, switches, cables de red, tarjetas de red y dispositivos de interconexión. Cada uno de estos elementos desempeña un papel específico en la facilitación de la transmisión de datos. Por ejemplo, los routers se encargan de dirigir el tráfico de datos entre diferentes redes, mientras que los switches permiten la interconexión de múltiples dispositivos dentro de una misma red local. Los cables de red, como los cables Ethernet y de fibra óptica, actúan como el medio físico para la transmisión de datos. Además, las tarjetas de red permiten que los dispositivos se conecten a la red, y los dispositivos de interconexión, como los repetidores y concentradores, amplían la cobertura y capacidad de la red. Entender el funcionamiento y la interacción de estos componentes es vital para garantizar una comunicación eficiente y segura en cualquier sistema de red.

Modelo OSI

El modelo de referencia OSI (Interconexión de Sistemas Abiertos u Open Systems Interconnection) es una arquitectura de red compuesta por 7 capas. Aunque es un modelo antiguo, su uso sigue siendo relevante.

Explicación de las Capas

  • Capa Física: Se ocupa de la transmisión de bits a través del medio físico de comunicación. Incluye todas las especificaciones mecánicas, eléctricas y de sincronización necesarias para su diseño.
  • Capa de Enlace de Datos: Tiene dos funciones principales: transformar el medio de transmisión en una línea libre de errores y gestionar el flujo de datos entre emisor y receptor. Divide los datos en tramas para su transmisión secuencial y controla el flujo de datos para evitar sobrecargar al receptor. En redes de difusión, también gestiona el acceso al canal compartido.
  • Capa de Red: Se encarga del enrutamiento de paquetes desde el origen hasta el destino. Puede utilizar tablas estáticas o dinámicas para adaptarse a los cambios en la red. Además, maneja la congestión y la calidad del servicio. En redes heterogéneas, resuelve problemas de direccionamiento, protocolos y tamaño de paquetes.
  • Capa de Transporte: Asegura la entrega correcta y eficiente de datos entre máquinas origen y destino, independientemente de los cambios en la tecnología de hardware. Proporciona servicios de conexión punto a punto o difusión de mensajes, y asegura la comunicación extremo a extremo.
  • Capa de Sesión: Permite establecer sesiones entre usuarios en diferentes ordenadores, controlando el envío, el flujo de datos y la sincronización.
  • Capa de Presentación: Se centra en la sintaxis y la semántica de la información transmitida, facilitando la comunicación entre computadoras con diferentes representaciones de datos. Define estructuras de datos abstractas y utiliza codificaciones estándar para la transmisión.
  • Capa de Aplicación: Contiene una variedad de protocolos que los usuarios necesitan frecuentemente. Un ejemplo es HTTP, utilizado en la World Wide Web para la transferencia de páginas web.

Caso de Aplicación

Imaginemos que una empresa está implementando una red de área local (LAN) para conectar varios departamentos. El modelo OSI puede ser utilizado para diseñar y organizar la red:

  • Capa Física: Define el tipo de cableado y medios de transmisión, como Ethernet o fibra óptica.
  • Capa de Enlace de Datos: Gestiona el acceso al medio y la detección de errores.
  • Capa de Red: Maneja el direccionamiento IP y el enrutamiento de paquetes.
  • Capa de Transporte: Asegura la entrega confiable de datos.
  • Capas de Sesión, Presentación y Aplicación: Establecen sesiones, convierten datos a formatos adecuados y proporcionan servicios de aplicaciones.

Modelo TCP/IP

El modelo TCP/IP se desarrolló para ARPANET e Internet, basándose en los protocolos TCP e IP. Este modelo se centra en la flexibilidad y la supervivencia de la red, permitiendo la conexión de múltiples redes sin problemas. A diferencia del modelo OSI, el TCP/IP no incluye capas explícitas para sesión y presentación, dejando estas funciones a discreción de las aplicaciones individuales.

Explicación de las Capas

  • Capa de Enlace: Define los enlaces necesarios para cumplir con los requisitos de una capa de interred sin conexión, operando a través de distintas redes. Funciona como una interfaz entre los hosts y los enlaces de transmisión.
  • Capa de Interred: Es fundamental para la arquitectura, permitiendo que los hosts envíen paquetes a cualquier red, incluso si el destino está en una red diferente. Define el formato de paquete y el protocolo IP, además del protocolo ICMP para mensajes de control.
  • Capa de Transporte: Permite la comunicación entre entidades pares en los nodos de origen y destino. Define dos protocolos principales: TCP, un protocolo confiable y orientado a la conexión, y UDP, un protocolo sin conexión y no confiable.
  • Capa de Aplicación: El modelo TCP/IP no incluye capas de sesión ni presentación. Las aplicaciones incorporan cualquier función necesaria de estas capas. Encima de la capa de transporte, se encuentran los protocolos de alto nivel como HTTP, FTP y SMTP.

Caso de Aplicación

Supongamos que una empresa necesita conectar su red local a Internet:

  • Capa de Aplicación: Incluye protocolos como HTTP, FTP y SMTP.
  • Capa de Transporte: Utiliza TCP para conexiones confiables y UDP para aplicaciones sin conexión.
  • Capa de Interred: Maneja el direccionamiento IP y el enrutamiento de paquetes.
  • Capa de Enlace: Se encarga de la transmisión de datos a través de medios físicos como Ethernet.

Conclusiones

Conocer las arquitecturas y redes es esencial para entender su diseño eficiente, rápido y sostenible. Aunque es mucha teoría, es crucial para identificar problemas y proponer soluciones efectivas.
